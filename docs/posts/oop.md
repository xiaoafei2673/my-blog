# 🧭 面向对象 OOP 导览

> 面向对象 OOP。以下是完整的 Java 面向对象知识体系树形图，作为学习的快速索引和导览。

```text
Java 面向对象知识体系
├── 编程范式 Programming Paradigm
│   ├── 面向过程 Procedural Programming：以函数和步骤为中心
│   │   ├── 核心特点：数据和操作相对分离
│   │   ├── 优点：简单直接，适合小程序、算法题
│   │   ├── 缺点：大型系统中代码容易散乱
│   │   └── 适合：脚本、算法、简单工具
│   ├── 面向对象 Object-Oriented Programming：以对象和职责为中心
│   │   ├── 核心特点：对象 = 数据 + 行为
│   │   ├── 优点：封装状态、复用代码、表达业务模型、支持扩展
│   │   ├── 缺点：设计不当会过度抽象、类层次复杂
│   │   └── 适合：业务系统、GUI、后端服务、复杂领域建模
│   └── 函数式 Functional Programming：以函数和不可变数据为中心
│       ├── 核心特点：函数作为一等公民，强调无副作用
│       ├── Java 支持：Lambda、Stream、函数式接口
│       └── 适合：集合处理、并发计算、声明式编程
│
├── Java 面向对象基础
│   ├── 类 Class：对象的模板
│   │   ├── 字段 Field：描述对象状态
│   │   ├── 方法 Method：描述对象行为
│   │   ├── 构造方法 Constructor：创建对象时初始化状态
│   │   ├── 代码块 Block：初始化逻辑
│   │   └── 内部类 Inner Class：定义在类内部的类
│   ├── 对象 Object：类的实例
│   │   ├── 创建方式：new ClassName()
│   │   ├── 存储位置：对象通常在 Heap 中
│   │   ├── 访问方式：通过引用 Reference 访问
│   │   ├── 生命周期：由 GC 垃圾回收管理
│   │   └── 易混淆点：变量保存的是对象引用，不是对象本体
│   ├── 引用 Reference
│   │   ├── 含义：指向对象的访问入口
│   │   ├── 示例：User user = new User()
│   │   ├── 特点：多个引用可以指向同一个对象
│   │   └── 易混淆点：对象相等 equals 与引用相等 == 不同
│   └── 包 Package
│       ├── 作用：组织类，避免命名冲突
│       ├── 示例：java.util、java.lang
│       └── 适合：模块化管理代码
│
├── 面向对象四大特性
│   ├── 封装 Encapsulation：隐藏内部细节，暴露受控接口
│   │   ├── 实现方式
│   │   │   ├── private 字段
│   │   │   ├── public 方法
│   │   │   ├── getter / setter
│   │   │   └── 构造方法校验
│   │   ├── 优点：保护数据、降低耦合、维护不变量
│   │   ├── 缺点：可能产生大量样板代码
│   │   ├── 适合：实体类、领域模型、服务类
│   │   └── 易混淆点：封装不是简单地给所有字段生成 getter/setter
│   ├── 继承 Inheritance：子类复用和扩展父类
│   │   ├── 关键字：extends
│   │   ├── 父类 Superclass
│   │   ├── 子类 Subclass
│   │   ├── 方法重写 Override
│   │   ├── super：访问父类构造方法或成员
│   │   ├── 优点：代码复用，表达 is-a 关系
│   │   ├── 缺点：父子类强耦合，层级过深难维护
│   │   ├── 适合：稳定的 is-a 层次结构
│   │   └── 易混淆点：has-a 关系不该用继承，应使用组合
│   ├── 多态 Polymorphism：同一抽象，不同实现
│   │   ├── 编译时多态
│   │   │   ├── 实现：方法重载 Overload
│   │   │   ├── 判断依据：方法参数列表
│   │   │   └── 发生时间：编译期
│   │   ├── 运行时多态
│   │   │   ├── 实现：继承 + 方法重写 + 父类引用指向子类对象
│   │   │   ├── 示例：Animal animal = new Dog()
│   │   │   ├── 底层：动态绑定 Dynamic Binding
│   │   │   ├── 优点：扩展性强，支持面向抽象编程
│   │   │   └── 适合：策略切换、插件化、框架扩展点
│   │   └── 易混淆点
│   │       ├── 能调用什么方法看引用类型
│   │       ├── 执行哪个重写方法看对象实际类型
│   │       └── 字段没有运行时多态，方法才有
│   └── 抽象 Abstraction：提取共性，屏蔽细节
│       ├── 抽象类 abstract class
│       │   ├── 特点：可以有字段、构造方法、普通方法、抽象方法
│       │   ├── 优点：适合定义共同模板
│       │   ├── 缺点：Java 单继承限制
│       │   ├── 适合：具有共同本质和共同状态的对象
│       │   └── 示例：Animal、Shape、BaseController
│       ├── 接口 interface
│       │   ├── 特点：定义能力规范
│       │   ├── 优点：支持多实现，解耦能力强
│       │   ├── 缺点：不适合承载复杂状态
│       │   ├── 适合：能力抽象、插件扩展、面向接口编程
│       │   └── 示例：Runnable、Comparable、Serializable
│       └── 易混淆点
│           ├── 抽象类强调 is-a
│           ├── 接口强调 can-do
│           └── 业务开发中通常优先依赖接口
│
├── Java 访问控制
│   ├── public：任意位置可访问
│   │   ├── 适合：对外 API
│   │   └── 风险：暴露过多会破坏封装
│   ├── protected：同包或子类可访问
│   │   ├── 适合：继承体系内部扩展
│   │   └── 易混淆点：不同包中必须通过子类访问
│   ├── default：同包可访问
│   │   ├── 适合：包内协作类
│   │   └── 特点：不写修饰符就是 default
│   └── private：仅当前类可访问
│       ├── 适合：内部状态、内部工具方法
│       └── 优点：封装性最强
│
├── Java 类成员机制
│   ├── 实例成员 Instance Member
│   │   ├── 实例字段：每个对象独有一份
│   │   ├── 实例方法：依赖具体对象调用
│   │   ├── 底层：对象存储在 Heap，引用在 Stack
│   │   └── 适合：对象状态和行为
│   ├── 静态成员 Static Member
│   │   ├── static 字段：类级别共享
│   │   ├── static 方法：不依赖对象
│   │   ├── 底层：类加载后与 Class 元数据关联
│   │   ├── 适合：工具方法、全局常量、共享计数
│   │   └── 易混淆点：static 方法不能直接访问实例字段
│   ├── final
│   │   ├── final 类：不能被继承
│   │   ├── final 方法：不能被重写
│   │   ├── final 变量：引用不能再指向别的对象
│   │   └── 易混淆点：final 引用对象的内部状态仍可能变化
│   └── 构造方法 Constructor
│       ├── 无参构造
│       ├── 有参构造
│       ├── 构造方法重载
│       ├── this()：调用本类其他构造方法
│       ├── super()：调用父类构造方法
│       └── 易混淆点：子类构造方法默认先调用父类无参构造
│
├── 类之间的关系
│   ├── 继承 Inheritance：is-a
│   │   ├── 示例：Dog extends Animal
│   │   ├── 优点：复用父类逻辑
│   │   ├── 缺点：耦合强
│   │   └── 适合：稳定的分类层次
│   ├── 实现 Implementation：can-do
│   │   ├── 示例：Dog implements RunnableAnimal
│   │   ├── 优点：能力抽象清晰
│   │   └── 适合：多能力组合
│   ├── 组合 Composition：has-a，强拥有关系
│   │   ├── 示例：Car has Engine
│   │   ├── 优点：比继承更灵活
│   │   ├── 适合：对象由多个部件组成
│   │   └── 原则：组合优于继承
│   ├── 聚合 Aggregation：has-a，弱拥有关系
│   │   ├── 示例：Team has Players
│   │   └── 特点：部分对象可独立存在
│   └── 依赖 Dependency：uses-a
│       ├── 示例：Service 使用 Repository
│       ├── 特点：临时使用
│       └── 适合：方法参数、局部变量
│
├── Object 根类
│   ├── Object：所有 Java 类的根父类
│   │   ├── toString()
│   │   │   ├── 作用：返回对象字符串表示
│   │   │   └── 常见重写：打印对象内容
│   │   ├── equals()
│   │   │   ├── 作用：判断对象逻辑相等
│   │   │   └── 易混淆点：== 判断引用是否相同
│   │   ├── hashCode()
│   │   │   ├── 作用：哈希容器中定位对象
│   │   │   └── 规则：重写 equals 通常也要重写 hashCode
│   │   ├── getClass()
│   │   │   ├── 作用：获取运行时 Class 对象
│   │   │   └── 关联：反射 Reflection
│   │   └── clone()
│   │       ├── 作用：对象复制
│   │       └── 易混淆点：浅拷贝和深拷贝不同
│
├── 面向对象设计原则 SOLID
│   ├── S：单一职责原则 SRP
│   │   ├── 含义：一个类只负责一类职责
│   │   └── 适合：降低类复杂度
│   ├── O：开闭原则 OCP
│   │   ├── 含义：对扩展开放，对修改关闭
│   │   └── 常用实现：接口 + 多态
│   ├── L：里氏替换原则 LSP
│   │   ├── 含义：子类对象应该可以替换父类对象
│   │   └── 易混淆点：子类不能破坏父类约定
│   ├── I：接口隔离原则 ISP
│   │   ├── 含义：接口要小而专，不强迫实现无关方法
│   │   └── 适合：拆分大接口
│   └── D：依赖倒置原则 DIP
│       ├── 含义：高层模块依赖抽象，不依赖具体实现
│       ├── 常用实现：interface + dependency injection
│       └── 适合：Spring、插件化、可测试代码
│
├── 常见设计模式中的面向对象
│   ├── 创建型模式
│   │   ├── 工厂模式 Factory：封装对象创建
│   │   ├── 抽象工厂 Abstract Factory：创建一组相关对象
│   │   ├── 建造者 Builder：分步骤构造复杂对象
│   │   └── 单例 Singleton：控制对象唯一实例
│   ├── 结构型模式
│   │   ├── 适配器 Adapter：转换接口
│   │   ├── 装饰器 Decorator：用组合增强对象能力
│   │   ├── 代理 Proxy：控制对象访问
│   │   └── 外观 Facade：提供统一入口
│   └── 行为型模式
│       ├── 策略 Strategy：接口 + 多态切换算法
│       ├── 模板方法 Template Method：抽象类定义流程骨架
│       ├── 观察者 Observer：对象状态变化通知订阅者
│       └── 责任链 Chain of Responsibility：多个处理器依次处理请求
│
└── Java 面向对象常见面试考点
    ├── 类和对象的区别
    │   ├── 类：模板
    │   ├── 对象：实例
    │   └── 易混淆点：引用变量不是对象本体
    ├── 封装、继承、多态、抽象
    │   ├── 封装：隐藏内部状态
    │   ├── 继承：复用和扩展父类
    │   ├── 多态：父类引用执行子类方法
    │   └── 抽象：提取共性、面向稳定接口
    ├── Overload vs Override
    │   ├── Overload：同类中，方法名相同，参数不同，编译期决定
    │   └── Override：父子类中，方法签名相同，运行期决定
    ├── 抽象类 vs 接口
    │   ├── 抽象类：共同本质，共同状态
    │   ├── 接口：共同能力，行为规范
    │   └── Java 类单继承，接口可多实现
    ├── == vs equals
    │   ├── ==：基本类型比值，引用类型比地址
    │   ├── equals：默认比地址，可重写为逻辑相等
    │   └── hashCode：与 equals 配合用于哈希集合
    ├── static
    │   ├── 属于类，不属于对象
    │   ├── 静态方法不能直接访问实例成员
    │   └── 常用于工具方法和共享常量
    ├── final
    │   ├── final 类不能继承
    │   ├── final 方法不能重写
    │   ├── final 变量不能重新赋值
    │   └── 易混淆点：final 对象引用不代表对象不可变
    └── 组合优于继承
        ├── 原因：继承耦合强，组合更灵活
        ├── 示例：Car has Engine，而不是 Car extends Engine
        └── 适合：业务系统中复杂对象协作
```
