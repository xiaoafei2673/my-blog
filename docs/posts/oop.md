## 📖 什么是面向对象？

**面向对象（Object-Oriented Programming, OOP）**即面向对象编程。Java 是一门以对象为核心的语言，强调对象的**封装**、**继承**、**多态**等特性。

::: info 💡 核心概念：什么是对象？
对象是将相关的数据和方法作为一个整体来看待。
简而言之：**对象 = 状态 + 行为 + 身份**
:::

通俗点来说，现实生活中的很多具体事物都可以被抽象成对象，比如人、车、仓库等(一个人，一辆车，一个仓库等都可以是一个对象)。每个对象通常都有自己的状态和行为：

- 📦 **状态（属性）**：对象拥有什么数据。比如人的姓名、年龄、性别。
- 🏃‍♂️ **行为（方法）**：对象能做什么事情。比如说话、移动、工作。

### 🆚 面向过程编程 vs 面向对象编程

| 编程范式 | 核心关注点 | 组织方式 |
| :--- | :--- | :--- |
| **面向过程编程** | “先做什么、再做什么” | 程序通常按照一个个**步骤**组织。 |
| **面向对象编程** | “有哪些对象、负责什么、如何协作” | 把对象作为程序的重要组织单位。每个对象封装状态和行为，承担各自职责，通过方法调用相互配合，最终完成业务功能。 |

## 🏗️ 从对象到类 (Class)

提到对象时，通常是指**对象个体**，而不是对象的集合。由对象可以延伸出“类”的概念。

::: tip 📝 什么是类？
**类 (Class)** 是对同一类事物（对象）的抽象描述，定义了对象的状态和行为。
- **包含**：成员变量（属性）+ 成员方法（行为）。
- **创建**：通过 `new 类名()` 来创建具体的对象。
:::

通俗点来说：**类是一个模板**（例如 `Student` 学生类），而**对象是一个具体的实例**（例如 `student1`、`张三` 等具体的学生个体）。类的成员变量规定了对象的状态，成员方法规定对象的行为。

### ❓ 为什么需要类？

- 📦 **封装**：封装对象的状态和行为，使代码更清晰、更易维护。
- ♻️ **复用**：复用对象的状态和行为，提高开发效率。
- 🧬 **共性**：定义对象的共性，方便后续的继承、多态，以及组合和聚合。

---

### 🆚 代码对比：有类 vs 无类

简单来说：**如果没有类**，当你需要描述多个学生对象时，就必须重复定义每个学生的状态，这会导致代码极度重复且难以维护。

```java
// ❌ 没有类：代码重复，维护困难
String name1 = "小明";
int age1 = 18;

String name2 = "小红";
int age2 = 19;
```

**有了类之后**，我们可以定义一个 `Student` 类，将属性和行为提取出来。每个学生对象只需作为这个类的实例即可。

```java
// ✅ 有类：只需定义一次模板
// 类名以大写字母开头，采用驼峰命名法
class Student {
    // 成员变量：描述对象的状态
    String name;
    int age;

    // 成员方法：描述对象的行为
    void study() {
        System.out.println(name + " is studying");
    }
}
```

定义好类模板后，我们就可以通过 `new` 关键字来批量创建独立的对象个体了：

```java
Student student1 = new Student(); // 创建学生对象 student1
student1.name = "Ethan";          // 赋值状态
student1.age = 20;
student1.study();                 // 执行行为，输出：Ethan is studying

Student student2 = new Student(); // 创建学生对象 student2
student2.name = "Lily";           // 赋值状态
student2.age = 21;
student2.study();                 // 执行行为，输出：Lily is studying
```

> **✨ 总结**：现在，`student1` 和 `student2` 都是独立的学生对象，它们的状态和行为是隔离且不同的。我们不用再重复定义多个对象的状态和行为，**只需要定义一次类模板，就可以描述无数个对象实例**。
